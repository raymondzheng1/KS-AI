## User Testing Protocol
📋  How to Run a Proper User Test
User testing is not asking "do you like it?" — that gets you polite answers, not useful ones. / Follow this protocol for each tester:

📋  User Testing Script
BEFORE: Set up / Tell the tester: "We are testing the tool, not you. There are no wrong answers. Please think out loud as you use it." / Give them a specific task: "Imagine you need to [specific use case]. Try using this tool to accomplish that." / DURING: Observe, don't help / Do NOT help them when they get stuck — their confusion is your most valuable data / Note exactly where they pause, hesitate, or make mistakes / Write down their exact words when they react to anything / AFTER: Ask these 5 questions / What would you use this for if you had it right now? / What confused you most? / What was the most useful part? / What one thing would you change? / Would you recommend this to a friend? Why or why not?

## Demo Day Presentation Template
🎤  Your 3-Minute Presentation Structure
Section 1: The Problem — 45 seconds / Open with a real scenario: "Imagine you are [user] and you face [problem]..." / Make the audience feel the problem before you show the solution / Section 2: Our Solution — 60 seconds / Show (not just tell) — live demo or screenshot walkthrough / Explain what AI does vs what the human does / Highlight the most impressive/surprising thing it can do / Section 3: What We Learned — 45 seconds / What was the hardest thing to build? / What did the AI do that surprised you? / What would you improve with more time? / Section 4: The Bigger Picture — 30 seconds / If this tool existed fully developed, what impact could it have? / Close with your team's key insight from this project
